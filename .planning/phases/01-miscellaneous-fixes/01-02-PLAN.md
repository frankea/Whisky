  <action>
  **Program model extension:**

  Add a `public let isClickOnce: Bool` property to Program. This is set during initialization:
  - In the existing `init(url:bottle:)`, set `isClickOnce = false` (default for .exe programs).
  - Add a new convenience initializer `init(appRefURL: URL, bottle: Bottle, displayName: String)` for ClickOnce apps:
    - `self.url = appRefURL` (the .appref-ms file is the stable launch artifact)
    - `self.isClickOnce = true`
    - `self.bottle = bottle`
    - The `name` computed property should return `displayName` for ClickOnce apps instead of `url.lastPathComponent`. To achieve this cleanly: add a `private let _displayName: String?` stored property (nil for regular programs, set for ClickOnce). The `name` computed property becomes: `_displayName ?? url.lastPathComponent`.
    - `self.peFile = nil` (no PE parsing for .appref-ms files)
    - Settings follow the same pattern (ProgramSettings loaded from settings folder)
    - `self.pinned` follows the same pin check pattern

  **ClickOnce app filtering in Bottle+Extensions:**

  In `updateInstalledPrograms()`:
  1. After scanning Program Files directories, add ClickOnce detection:
     ```swift
     // Detect ClickOnce applications
     let clickOnceApps = ClickOnceManager.shared.detectAppRefFile(in: self, wineUsername: wineUsername)
     for appRefURL in clickOnceApps {
         let displayName = ClickOnceManager.shared.displayName(for: appRefURL)
         let program = Program(appRefURL: appRefURL, bottle: self, displayName: displayName)
         programs.append(program)
     }
     ```

  2. Filter OUT ClickOnce cache executables from the regular program scan. ClickOnce cache lives under `drive_c/users/{username}/AppData/Local/Apps/2.0/`. Add a check in the .exe scanning loop:
     ```swift
     // Skip ClickOnce cache executables (noisy internal artifacts)
     guard !url.path.contains("/Apps/2.0/") else { continue }
     ```
     Place this guard after the `guard !url.hasDirectoryPath, url.pathExtension == "exe"` line.

  3. Import WhiskyKit at the top of Bottle+Extensions.swift (already imported).

  **Note:** The `wineUsername` property is already available on Bottle in Bottle+Extensions.swift as a @MainActor computed property.
  </action>
  <verify>
  `xcodebuild -project Whisky.xcodeproj -scheme Whisky -configuration Debug build` succeeds. A bottle with ClickOnce apps would show them in the programs list (manual verification in later checkpoint).
  </verify>
  <done>
  Program has isClickOnce flag and ClickOnce-specific initializer. updateInstalledPrograms detects ClickOnce .appref-ms files and filters out cache executables.

  2. "Remove ClickOnce App" button: Removes the .appref-ms file and optionally the ClickOnce cache. Use a destructive button style. After removal, trigger a reload of the program list.

  For non-ClickOnce programs, the existing menu items remain unchanged.

  Wrap the ClickOnce-specific actions in `if program.isClickOnce { ... }` within the context menu Section.

  **BottleView launch support:**

This should work because Wine handles .appref-ms files via its ClickOnce/DCOM infrastructure.
