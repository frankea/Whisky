**GameEntryDetailView.swift:**
Create `GameEntryDetailView` struct:

**Properties:**
- `let entry: GameDBEntry`
- `@ObservedObject var bottle: Bottle`
- `@State private var selectedVariant: GameConfigVariant?`
- `@State private var showPreviewSheet: Bool = false`
- `@State private var stalenessResult: StalenessResult?`
- `@State private var toast: ToastData?`

**On appear:**
- Set selectedVariant to entry.defaultVariant (auto-select for machine per CONTEXT.md -- for v1, use defaultVariant; full machine-aware auto-selection is a refinement)

**Body layout -- ScrollView with Form sections matching CONTEXT.md detail view:**

**GameConfigPreviewSheet.swift:**
Create `GameConfigPreviewSheet` struct:

**Properties:**
- `let entry: GameDBEntry`
- `let variant: GameConfigVariant`
- `@ObservedObject var bottle: Bottle`
- `let programURL: URL?` -- nil if applying at bottle level
- `@Environment(\.dismiss) private var dismiss`
- `@State private var changes: [ConfigChange] = []`
- `@State private var missingVerbs: [String] = []`
- `@State private var isApplying: Bool = false`
- `@State private var applyError: String?`
- `@State private var stalenessResult: StalenessResult?`
- `@State private var includeWinetricks: Bool = true` -- checkbox to include/exclude verb installs
- `@AppStorage("gameConfigSkipPreview") private var skipPreview: Bool = false` -- "Don't show again"
- `@Binding var toast: ToastData?`

**On appear:**
1. Load changes via `GameConfigApplicator.previewChanges(variant:, bottle:)`
2. Load installed verbs via `Winetricks.loadInstalledVerbs(for: bottle)` (async), then compute `missingVerbs = GameConfigApplicator.pendingWinetricksVerbs(variant:, installedVerbs:)`
3. Check staleness via `StalenessChecker.check(testedWith:)`

**Body layout -- Form in a sheet:**

**Staleness warning** (if stale): Yellow banner with warning icon and message. "Apply anyway" action below.

**Apply target section:**
- Show "Apply to: [bottle name]" with option to select program-level if programURL exists
- Toggle for "Apply at program level" vs "Apply at bottle level" if context allows

**Changes section -- grouped by ConfigChange.category:**
- For each category (Graphics, Performance, DLL Overrides, etc.), show a Section:
  - Each change as HStack: settingName | currentValue (strikethrough, red) -> newValue (green)
  - High-impact changes marked with exclamationmark.triangle icon

**Winetricks section** (if variant has winetricks verbs):
- List of required verbs with checkmark for already-installed, xmark for missing
- Toggle `includeWinetricks` -- "Install Required Components" checkbox
- If unchecked, show note: "Config will be marked as 'Incomplete (missing dependencies)'"
- Note: "Winetricks installations modify the Wine prefix and cannot be undone"

**Restart note** (if changes include graphics backend):
- "These changes will take effect on next launch"

**Bottom bar:**
- "Don't show preview again" checkbox bound to `skipPreview`
- Cancel button
- "Apply Configuration" button (primary, async action):
  1. Set isApplying = true
  2. Call `GameConfigApplicator.apply(entry:, variant:, to: bottle, programURL:)`
  3. If includeWinetricks and missingVerbs not empty, note them in snapshot.installedVerbs (actual verb installation is left to the user via existing Winetricks UI -- record intent but don't auto-install per CONTEXT.md "explicit Install Required Components action")
  4. On success: dismiss sheet, set toast to success message with "Undo" action
  5. On failure: show applyError inline

**Undo toast integration:**
The success toast should include an "Undo" button that loads the snapshot and calls `GameConfigApplicator.revert(bottle:, snapshot:)`.

**GameConfigurationView.swift update:**
Wire the NavigationLink destination in GameConfigurationView to point to the real `GameEntryDetailView` instead of the placeholder Text. Pass bottle through.
