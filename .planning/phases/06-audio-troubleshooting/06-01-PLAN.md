  <action>
    Create AudioDeviceMonitor.swift -- a class that wraps CoreAudio's C API for device enumeration, property queries, and change listeners. Import CoreAudio and Foundation.

    **Class design**: `public final class AudioDeviceMonitor: @unchecked Sendable`. Use `@unchecked Sendable` because CoreAudio callbacks dispatch to DispatchQueue.main. Store a private logger via `os.log` Logger(subsystem: "com.isaacmarovitz.Whisky", category: "AudioDeviceMonitor").

    **Public API**:
    - `init()` -- no arguments
    - `func defaultOutputDevice() -> AudioDeviceInfo?` -- queries the system default output device using kAudioHardwarePropertyDefaultOutputDevice, then calls internal queryDeviceInfo
    - `func allOutputDevices() -> [AudioDeviceInfo]` -- queries all devices via kAudioHardwarePropertyDevices, filters to those with output channels > 0
    - `func startListening(onChange: @escaping @Sendable (AudioDeviceChangeEvent) -> Void)` -- registers CoreAudio property listener for default output device changes. Use AudioObjectAddPropertyListenerBlock with DispatchQueue.main as the queue parameter (per research pitfall #1). Store the block reference for removal.
    - `func stopListening()` -- removes the registered listener via AudioObjectRemovePropertyListenerBlock. Call in deinit as well (per research pitfall #2).
    - `deinit` -- calls stopListening()

    **Internal helpers** (private):
    - `queryDeviceInfo(_ deviceID: AudioDeviceID) -> AudioDeviceInfo?` -- queries device name (kAudioObjectPropertyName), transport type (kAudioDevicePropertyTransportType), sample rate (kAudioDevicePropertyNominalSampleRate), output channel count (kAudioDevicePropertyStreamConfiguration with kAudioObjectPropertyScopeOutput). See research code examples for exact AudioObjectPropertyAddress patterns.
    - Use `kAudioObjectPropertyElementMain` (NOT the deprecated kAudioObjectPropertyElementMaster per research).
    - For device name query: use CFString and AudioObjectGetPropertyData.
    - For transport type: UInt32 -> AudioTransportType(rawValue:).
    - For sample rate: Float64 from kAudioDevicePropertyNominalSampleRate.
    - For channel count: Use kAudioDevicePropertyStreamConfiguration with AudioBufferList (see research code example for queryOutputChannelCount). Allocate UnsafeMutableRawPointer, defer deallocate, sum mNumberChannels across buffers.

    **Listener implementation detail**:
    - Store `listenerBlock` as Optional closure and `listenerAddress` as Optional AudioObjectPropertyAddress for proper removal.
    - In the listener callback, construct AudioDeviceChangeEvent with eventType .defaultOutputChanged, deviceName from defaultOutputDevice()?.name, transportType from defaultOutputDevice()?.transportType.
    - Handle the case where defaultOutputDevice() returns nil (device removed) -- use "Unknown" name and .unknown transport.

    **Error handling**: Log errors via logger.error but never throw from query methods -- return nil/empty arrays. CoreAudio errors are status codes (OSStatus); check `status == noErr`.
