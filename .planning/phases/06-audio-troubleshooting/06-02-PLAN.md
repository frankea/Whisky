       - `.coreaudio` -- Force CoreAudio driver. displayName: "CoreAudio". registryValue: "coreaudio"

       - `.defaultPreset` -- Wine default. displayName: localized "config.audio.latency.default". helBuflenValue: 65536
       - `.lowLatency` -- Low latency. displayName: localized "config.audio.latency.low". helBuflenValue: 512
       - `.stable` -- Larger buffer for stability. displayName: localized "config.audio.latency.stable". helBuflenValue: 131072
       Include `displayName: String` and `helBuflenValue: Int` computed properties.

       - `.followSystem` -- Follow macOS default (recommended). displayName: localized "config.audio.device.followSystem"
       - `.pinned` -- Pin to specific device. displayName: localized "config.audio.device.pinned"
       Include `displayName: String` computed property.

    `BottleAudioConfig: Codable, Equatable` struct with:
    - `var audioDriver: AudioDriverMode = .auto`
    - `var latencyPreset: AudioLatencyPreset = .defaultPreset`
    - `var outputDeviceMode: OutputDeviceMode = .followSystem`

Also promote the `RegistryType` enum from internal to `public` (it needs to be accessible from WineAudioRegistry). These methods are already inside `public extension Wine {}` so removing `private` makes them internal to the module. This allows WineAudioRegistry.swift (in the same module) to call them.

    **WineAudioRegistry.swift**: Create a new file with `import Foundation` and a `public extension Wine {}` block containing:

    1. Private enum `AudioRegistryKey: String` with:
       - `case drivers = "HKCU\\Software\\Wine\\Drivers"`
       - `case directSound = "HKCU\\Software\\Wine\\DirectSound"`
       - `case mmDevices = "HKLM\\Software\\Microsoft\\Windows\\CurrentVersion\\MMDevices"`

    2. `@MainActor static func readAudioDriver(bottle: Bottle) async throws -> String?`
       -- calls queryRegistryKey(bottle:key:name:type:) with drivers key, name "Audio", type .string

    3. `@MainActor static func setAudioDriver(bottle: Bottle, driver: AudioDriverMode) async throws`
       -- if driver.registryValue is non-nil, calls addRegistryKey with drivers key, name "Audio", data: value, type .string
       -- if driver is .auto (registryValue == nil): call runWine with ["reg", "delete", "HKCU\\Software\\Wine\\Drivers", "/v", "Audio", "/f"] to remove the key (let Wine auto-detect). Wrap in do/catch since the key might not exist.

    4. `@MainActor static func readDirectSoundBuffer(bottle: Bottle) async throws -> Int?`
       -- calls queryRegistryKey with directSound key, name "HelBuflen", type .string
       -- parse result as Int

    5. `@MainActor static func setDirectSoundBuffer(bottle: Bottle, helBuflen: Int) async throws`
       -- calls addRegistryKey with directSound key, name "HelBuflen", data String(helBuflen), type .string

    6. `@MainActor static func resetAudioState(bottle: Bottle) async throws`
       -- Deletes the MMDevices registry subtree: runWine(["reg", "delete", mmDevices key, "/f"], bottle:). Wrap in do/catch.
       -- Then kills wineserver to force fresh restart: calls Wine.killBottle(bottle:) (existing method).
       -- Log actions via Logger.

    **BottleSettings.swift -- populateAudioLayer**:
    Add a new method `public func populateAudioLayer(builder: inout EnvironmentBuilder)` in the EnvironmentBuilder Layer Populators section. This method:

    - Note: Audio driver and buffer settings are applied via Wine registry (WineAudioRegistry), NOT environment variables. The populateAudioLayer method is reserved for future audio-related env vars (e.g., WINEDEBUG channels for audio debugging).
    - For now, this is a placeholder method that the EnvironmentBuilder cascade can call, following the pattern of other populate methods. It ensures the audio layer integration point exists.
    - Add a comment explaining that audio configuration is registry-backed (unlike graphics which uses env vars).
