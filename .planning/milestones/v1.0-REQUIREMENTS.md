# Requirements: Whisky Upstream Issue Resolution

**Defined:** 2026-02-08
**Core Value:** Every tracking issue (#40-#50) has a concrete response — code fix, configuration UI, or in-app guidance

## v1 Requirements

Requirements for this milestone. Each maps to roadmap phases.

### Configuration Foundation

- [x] **CFGF-01**: Environment variable precedence is explicit and documented across Wine defaults, macOS fixes, bottle settings, launcher presets, game profiles, program settings, and user overrides
- [x] **CFGF-02**: Conflicting override strategies in MacOSCompatibility, BottleSettings, and LauncherPresets are refactored into a single EnvironmentBuilder
- [x] **CFGF-03**: ProgramSettings supports DLL override entries (WINEDLLOVERRIDES per-program)
- [x] **CFGF-04**: ProgramSettings supports winetricks verb tracking per-program

### Graphics Configuration

- [x] **GFXC-01**: User can toggle between D3DMetal, DXVK, and wined3d graphics backends per bottle
- [x] **GFXC-02**: Graphics settings UI uses tiered Simple/Advanced layout to avoid overwhelming users
- [x] **GFXC-03**: User can configure DXVK-specific settings (async shader compilation, HUD) per bottle
- [x] **GFXC-04**: User can override graphics backend per program (overrides bottle-level setting)
- [x] **GFXC-05**: DXVK config file management with user-editable settings exposed in UI

### Audio Troubleshooting

- [x] **AUDT-01**: User can view CoreAudio device detection status and sample rate information in-app
- [x] **AUDT-02**: User can access in-app audio troubleshooting guide with step-by-step remediation
- [x] **AUDT-03**: User can configure up to 3-4 audio-related Wine settings per bottle (FAudio/dsound selection, sample rate override)
- [x] **AUDT-04**: Audio diagnostics panel detects Bluetooth device disconnects and sample rate mismatches

### Process Management

- [x] **PROC-01**: ProcessRegistry tracks wineserver PID per WINEPREFIX for accurate bottle-level lifecycle awareness
- [x] **PROC-02**: Orphan Wine processes are detected and user is prompted to clean them up
- [x] **PROC-03**: Wine processes auto-terminate when user closes a bottle or quits Whisky
- [x] **PROC-04**: RunningProcessesView shows live process status with wineserver-aware tracking
- [x] **PROC-05**: User can force-kill individual Wine processes or all processes in a bottle

### Stability & Diagnostics

- [x] **STAB-01**: WINEDEBUG error patterns are parsed and classified from Wine output (access violations, GPU errors, DLL load failures)
- [x] **STAB-02**: Known crash patterns are detected and matched to suggested remediation steps
- [x] **STAB-03**: GPU crash pattern detection provides specific guidance for IOMFB and Metal errors
- [x] **STAB-04**: Diagnostic reports include classified error summary alongside raw logs

### Game Compatibility

- [x] **GAME-01**: Bundled JSON database contains known-good configurations for top 20-50 commonly reported games
- [x] **GAME-02**: User can view per-game troubleshooting notes and recommended settings from the compatibility database
- [x] **GAME-03**: User can apply recommended game configuration with one click (sets env vars, DLL overrides, winetricks)
- [x] **GAME-04**: Compatibility entries include version tuples (Wine version, macOS version) with staleness warnings

### Guided Troubleshooting

- [x] **TRBL-01**: Data-driven troubleshooting engine loads symptom/solution mappings from JSON
- [x] **TRBL-02**: User can navigate troubleshooting guides for graphics issues (black screen, artifacts, low FPS)
- [x] **TRBL-03**: User can navigate troubleshooting guides for audio issues (crackling, missing sound, stuttering)
- [x] **TRBL-04**: User can navigate troubleshooting guides for launcher issues (Steam, EA, Rockstar, Epic)
- [x] **TRBL-05**: Troubleshooting guides include automated diagnostic checks (setting verification, file existence, log pattern matching)

### Remaining Launcher Issues

- [x] **LNCH-01**: Steam download stall detection with guidance to change download region or restart
- [x] **LNCH-02**: macOS 15.4+ Steam compatibility documented with workaround guidance
- [x] **LNCH-03**: EA App troubleshooting guidance with known working configuration
- [x] **LNCH-04**: Rockstar Games Launcher troubleshooting guidance with LauncherPatcher.exe recommendation

### Remaining Controller Issues

- [x] **CTRL-01**: Controller detection persists across sessions without requiring Mac reboot
- [x] **CTRL-02**: Microphone permission request flow implemented for apps requiring mic access
- [x] **CTRL-03**: Per-controller mapping improvements for DualShock 4, DualSense, and Xbox controllers
- [x] **CTRL-04**: Retina mode mouse position scaling fix for fullscreen games

### Remaining Installation Issues

- [x] **INST-01**: In-app dependency installation guide for DirectX, .NET Framework, and vcruntime
- [x] **INST-02**: Permission flow improvements to reduce repeated volume access prompts
- [x] **INST-03**: Dependency installation status tracking showing what's installed per bottle

### Remaining UI/UX

- [x] **UIUX-01**: GPTK update dialog simplified to remove unnecessary extra confirmation step
- [x] **UIUX-02**: Desktop shortcut creation handles paths with spaces correctly
- [x] **UIUX-03**: WhiskyCmd improvements for reliable app launching and output display
- [x] **UIUX-04**: Build Version and Retina Mode show actual values instead of "N/A"

### Feature Requests

- [x] **FEAT-01**: User can toggle App Nap management per bottle for improved stability
- [x] **FEAT-02**: User can duplicate an existing bottle for experimentation
- [x] **FEAT-03**: User can change display resolution for Windows apps
- [x] **FEAT-04**: Console output persists after program termination for diagnostic review

### Miscellaneous

- [x] **MISC-01**: ClickOnceManager handles ClickOnce application deployment and management
- [x] **MISC-02**: Clipboard handling prevents freezes in multiplayer games (Risk of Rain 2 pattern)
- [x] **MISC-03**: Temp file cleanup runs automatically on bottle close or app quit
- [x] **MISC-04**: Wine process cleanup automation prevents resource leaks from zombie processes

## Out of Scope

| Feature | Reason |
|---------|--------|
| Wine/GPTK source code patches | Whisky consumes Wine binaries, doesn't build them |
| DXMT compilation/bundling | Separate project; include if available upstream |
| Anti-cheat compatibility (Denuvo, EAC) | Kernel-level, not solvable in app layer |
| Kernel-mode driver support | macOS limitation |
| Game-specific Wine patches | Wine/Proton upstream responsibility |
| Full community compat database | Decays fast, tiny macOS contributor base; use curated JSON instead |
| Mobile app | Web/desktop focus for this milestone |
