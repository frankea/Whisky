    Data types (all Codable, Sendable):

    CrashCategory: enum with 7 cases matching the taxonomy:
      .coreCrashFatal, .graphics, .dependenciesLoading, .prefixFilesystem,
      .networkingLaunchers, .antiCheatUnsupported, .otherUnknown
      Each case has displayName (String) and sfSymbol (String) properties.

    PatternSeverity: enum { critical, error, warning, info } with Comparable conformance.

    ConfidenceTier: enum { high, medium, low } with:
      - init(score: Double) mapping: >= 0.8 = .high, >= 0.5 = .medium, else .low
      - displayName property

    CrashPattern: struct { id, category, severity, confidence (Double 0-1),
      substringPrefilter (String?), regex (String), tags ([String]),
      captureGroups ([String]?), remediationActionIds ([String]?) }
      Has a transient compiledRegex property (not Codable).
      match(line:) method: check prefilter -> compile regex lazily -> evaluate -> return PatternMatch? with captures.

    PatternMatch: struct { captures: [String] }

    DiagnosisMatch: struct { pattern: CrashPattern, lineIndex: Int, lineText: String, captures: [String] }

    RemediationAction: struct { id, title, description, category (CrashCategory),
      actionType (enum: changeSetting, installVerb, switchBackend, informational),
      risk (enum: low, medium, high),
      settingKeyPath (String?), settingValue (String?),
      winetricksVerb (String?),
      whatWillChange (String), undoPath (String), appliesNextLaunch (Bool) }

    CrashDiagnosis: struct { matches: [DiagnosisMatch],
      categoryCounts: [CrashCategory: Int],
      primaryCategory: CrashCategory?,
      primaryConfidence: ConfidenceTier?,
      headline: String?,
      exitCode: Int32?,
      applicableRemediationIds: [String] }
      Has computed property isEmpty.
      Has method remediations(from:) that resolves IDs to RemediationAction array sorted by confidence then risk.

    PatternLoader: enum (caseless, like GPUDetection pattern)
      static loadPatterns(from: URL) throws -> [CrashPattern]
      static loadRemediations(from: URL) throws -> [String: RemediationAction]
      static loadDefaults() -> ([CrashPattern], [String: RemediationAction])
        Uses Bundle.module for SPM resource access.
        Debug: fatalError on invalid JSON. Release: return empty arrays.

  <behavior>
    CrashClassifier is a Sendable struct initialized with arrays of CrashPattern and a dictionary of RemediationAction.
    It exposes a single public method: classify(log: String, exitCode: Int32?) -> CrashDiagnosis.

    Classification pipeline:
    1. Split log into lines
    2. For each line, iterate patterns:
       a. If substringPrefilter is set and line does NOT contain it, skip (fast path)
       b. Otherwise, evaluate compiled Regex against the line
       c. On match, create DiagnosisMatch with pattern ref, lineIndex, captures
    3. After scanning, aggregate matches by category
    4. Compute primary diagnosis: highest severity + confidence match
    5. Build CrashDiagnosis with: matches, categoryCounts, primaryCategory, exitCode, applicable remediationActionIds

    Test cases (input -> expected output):

    Empty log, exit 0:
      -> CrashDiagnosis with empty matches, no primary category

    DLL load failure line:
      "0024:err:module:import_dll Library MSVCR100.dll (which is needed by L\"game.exe\") not found"
      -> match category: .dependenciesLoading, confidence: .high, captures: ["MSVCR100.dll"]

    Access violation line:
      "0080:err:seh:NtRaiseException Unhandled exception code c0000005 flags 0 addr 0x7b012345"
      -> match category: .coreCrashFatal, confidence: .high

    Wine page fault:
      "wine: Unhandled page fault on read access to 0x00000000 at address 0x7b012345"
      -> match category: .coreCrashFatal, confidence: .high

    GPU device lost (DXVK):
      "err:d3d11:device_lost D3D11 device lost (DXGI_ERROR_DEVICE_REMOVED)"
      -> match category: .graphics, confidence: .medium

    Anti-cheat signature:
      "EasyAntiCheat.exe: Fatal error - EasyAntiCheat is not installed"
      -> match category: .antiCheatUnsupported, confidence: .high

    .NET CLR issue:
      "0024:err:module:import_dll Library mscoree.dll (which is needed by L\"app.exe\") not found"
      -> match category: .dependenciesLoading, confidence: .high, captures: ["mscoree.dll"]

    Prefix corruption:
      "wine: could not load user32.dll, status c0000135"
      -> match category: .prefixFilesystem, confidence: .medium

    Network TLS error:
      "0024:err:winhttp:request_send_request secure connection failed"
      -> match category: .networkingLaunchers, confidence: .medium

    Multiple matches in one log -> sorted by confidence DESC then severity DESC

    Prefilter optimization:
      Pattern with substringPrefilter "import_dll" should NOT evaluate regex against line "fixme:d3d:something"

    WineDebugPreset: enum with 4 cases { normal, crash, dllLoad, verbose }
      Each has winedebugValue, displayName, and description computed properties.
      CaseIterable, Codable, Sendable.
  </behavior>
  <implementation>
    CrashClassifier: struct (Sendable) { patterns, remediations }
      init(patterns:remediations:) compiles all regex patterns at init time.
      classify(log:exitCode:) -> CrashDiagnosis

    WineDebugPreset: enum { normal, crash, dllLoad, verbose }

      description: String computed property (explains what it enables)

      - Prefilter optimization (verify regex not called when prefilter misses)
      - WineDebugPreset value assertions
