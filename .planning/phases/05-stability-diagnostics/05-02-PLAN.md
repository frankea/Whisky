    Create three pure-logic types in WhiskyKit/Diagnostics/:

    **DiagnosisHistory.swift:**
    - `DiagnosisHistoryEntry` struct (Codable, Sendable): timestamp (Date), logFileRef (String), primaryCategory (CrashCategory), confidenceTier (ConfidenceTier), topSignatures ([String], max 3), remediationCardIds ([String]), wineDebugPreset (WineDebugPreset?), bottleIdentifier (String), programPath (String)
    - `DiagnosisHistory` struct (Codable, Sendable): entries ([DiagnosisHistoryEntry]), static maxEntries = 5
    - mutating func append(_ entry:) with FIFO eviction (remove oldest when count > maxEntries)
    - mutating func clear()
    - computed var isEmpty: Bool
    - static func load(from url: URL) -> DiagnosisHistory (decode plist, return empty on failure)
    - func save(to url: URL) throws (encode as plist)
    - Follow ProgramSettings persistence pattern (PropertyListEncoder/Decoder)

    **RemediationTimeline.swift:**
    - `RemediationTimelineEntry` struct (Codable, Sendable): timestamp (Date), actionId (String), actionTitle (String), beforeValue (String?), afterValue (String?), bottleIdentifier (String), programPath (String)
    - `RemediationTimeline` struct (Codable, Sendable): entries ([RemediationTimelineEntry]), static maxEntries = 10
    - mutating func record(_ entry:) with FIFO eviction
    - mutating func clear()
    - computed var isEmpty: Bool
    - Same load/save pattern as DiagnosisHistory

    **Redactor.swift:**
    - `Redactor` enum (caseless, Sendable) following GPUDetection pattern
    - static let sensitiveKeyPatterns = ["TOKEN", "KEY", "SECRET", "PASSWORD", "AUTH"]
    - static func redactHomePaths(_ text: String) -> String
      Uses `FileManager.default.homeDirectoryForCurrentUser.path` to detect actual home dir pattern, replaces `/Users/<username>` with `/Users/<redacted>` globally
    - static func redactEnvironment(_ env: [String: String], includeSensitive: Bool = false) -> [String: String]
      When includeSensitive is false (default): redact home paths in values AND remove keys matching sensitiveKeyPatterns (case-insensitive contains check)
      When includeSensitive is true: only redact home paths (keep sensitive keys)
    - static func redactLogText(_ text: String) -> String
      Redact home paths throughout log text

    GPL v3 file header on all files. All types Sendable and Codable.

  <action>
    Create the diagnostic export engine and its tests:

    **DiagnosticExporter.swift:**
    - `DiagnosticExporter` enum (caseless, Sendable)
    - `ExportOptions` struct: includeSensitiveDetails (Bool, default false), includeRemediationHistory (Bool, default true), includeFullLog (Bool, default true), tailLineCount (Int, default 500)

    **ZIP export method:**
    static func exportZIP(diagnosis: CrashDiagnosis, bottle: Bottle, program: Program, logFileURL: URL?, history: DiagnosisHistory?, timeline: RemediationTimeline?, options: ExportOptions) async -> URL

    Steps:
    1. Create temp directory with UUID name
    2. Generate and write each file:
       - `report.md`: Human-readable Markdown summary (headline diagnosis, confidence tier, category counts, matched patterns, suggested remediations, environment summary). Apply Redactor unless includeSensitiveDetails.
       - `crash.json`: JSONEncoder output of CrashDiagnosis (matches, categories, scores)
       - `env.json`: Resolved environment from EnvironmentBuilder.resolve() with provenance, passed through Redactor.redactEnvironment(includeSensitive: options.includeSensitiveDetails)
       - `bottle-settings.json`: Bottle settings summary (name, winVersion, graphicsBackend, key config flags)
       - `program-settings.json`: Program settings and overrides summary
       - `wine.log`: Full log file copy (if includeFullLog and logFileURL provided)
       - `wine.tail.log`: Last N lines of log (use StabilityDiagnostics.tailOfLogFile pattern, but with configurable line count)
       - `system.json`: macOS version, architecture, Whisky version, WhiskyWine version, GPU info (reuse StabilityDiagnostics.generateSystemInfo pattern)
       - `remediation-history.json`: RemediationTimeline entries (if includeRemediationHistory and timeline provided)
    3. Run /usr/bin/ditto to create ZIP from temp directory
    4. Clean up temp directory
    5. Return ZIP URL

    ZIP filename: `Whisky-Diagnostics-<bottleName>-<programName>-<YYYYMMDD-HHmmss>.zip`
    (sanitize bottle/program names: replace non-alphanumeric with hyphens)

    **Markdown copy method:**
    static func generateMarkdownReport(diagnosis: CrashDiagnosis, bottle: Bottle, program: Program, options: ExportOptions) async -> String

    Produces a single Markdown block suitable for pasting into GitHub issues:
    - Header: "## Whisky Diagnostic Report"
    - Sections: System Info, Bottle Config, Diagnosis Summary (headline + confidence + category counts), Matched Patterns (top 5), Suggested Remediations, Environment Keys (keys only, no values)
    - Always redacted (no sensitive details in clipboard copy)
    - Compact format (no raw log -- just summary)

    - testMarkdownReportRedacted: verify no home paths leak

    Note: ZIP export test is impractical in unit tests (requires /usr/bin/ditto, file system). Test the content generation methods instead.
