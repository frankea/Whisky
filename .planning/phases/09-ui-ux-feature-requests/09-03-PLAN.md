  <name>Task 2: Per-program display overrides in ProgramOverrideSettingsView</name>
  <files>
    Whisky/Views/Programs/ProgramOverrideSettingsView.swift

  <action>
1. **Add Display override group** to ProgramOverrideSettingsView following the existing inherit/override pattern used for Graphics, Input, and other groups:
   - New section: "Display" with inherit/override toggle
   - When overriding:
     - Toggle: "Use Virtual Desktop" bound to `overrides.virtualDesktopEnabled`
     - Picker: Resolution preset bound to `overrides.resolutionPreset`
     - Custom width/height fields when preset is `.custom`
   - Copy-on-enable: when switching from inherit to override, copy current bottle values per the Phase 2 pattern
   - Nil-on-disable: when switching back to inherit, set all display override fields to nil

2. **Wire resolution overrides into EnvironmentBuilder/launch flow:**
   - In the program launch path, when virtual desktop overrides exist in ProgramOverrides, apply them via registry before launch. Since Wine registry keys are global to the prefix (not per-process), the approach is: write registry key, launch program, optionally restore after exit. For simplicity in V1: just let the per-program override write the registry key at launch time and leave it (the user expects "changes take effect on next launch" semantics).
   - Alternatively, use Wine's `explorer /desktop=ProgramName,WxH` command-line approach for per-process virtual desktop. This is cleaner: modify the Wine command arguments in `Wine.runProgram()` to prepend `explorer /desktop=<name>,<WxH>` when the program has virtual desktop override enabled. This avoids global registry mutation.
   - Recommendation: Use the `explorer /desktop=` command-line approach for per-program overrides (does not require registry writes, is per-process). Use registry approach for bottle-level default (written once when user toggles).

3. Add localization keys: `programOverride.display`, `programOverride.display.info`.
  </action>
