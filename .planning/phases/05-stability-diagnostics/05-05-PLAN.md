  <action>
    **DiagnosticExportSheet.swift:**
    A sheet view presented from DiagnosticsView for exporting diagnostic reports.

    Layout:
    - Title: "Export Diagnostic Report"
    - Filename preview: "Whisky-Diagnostics-{bottle}-{program}-{timestamp}.zip"
    - Checkboxes (per locked decisions):
      - "Include sensitive details (recommended only when sharing privately)" -- Toggle, OFF by default
      - "Include remediation history" -- Toggle, ON by default (labeled inversely from decision: "Don't include" = checkbox OFF means include)
      - "Include full log file" -- Toggle, ON by default
    - Two action buttons:
      - "Save ZIP..." -- presents NSSavePanel, calls DiagnosticExporter.exportZIP() with selected options, moves ZIP to user-chosen location
      - "Copy Report to Clipboard" -- calls DiagnosticExporter.generateMarkdownReport(), copies to NSPasteboard. Shows brief success toast.
    - "Cancel" button to dismiss

    State:
    @State includeSensitive = false
    @State includeRemediationHistory = true
    @State includeFullLog = true
    @State isExporting = false (shows ProgressView during export)

    Map toggles to ExportOptions:
    ```swift
    ExportOptions(
        includeSensitiveDetails: includeSensitive,
        includeRemediationHistory: includeRemediationHistory,
        includeFullLog: includeFullLog
    )
    ```

    NSSavePanel configuration:
    - allowedContentTypes: [.zip]
    - nameFieldStringValue: sanitized filename
    - After export, move generated ZIP from temp to user-chosen URL

    **DiagnosisHistoryView.swift:**
    A view showing the per-program diagnosis history (last 5 entries).

    Layout:
    - Section header: "Crash Diagnosis History"
    - If history is empty: "No crash diagnoses recorded" message with "Analyze last run" button
    - If history has entries: List of entries, each showing:
      - Primary category icon + name
      - Confidence tier badge (colored)
      - Timestamp (relative: "2 hours ago" via Text(date, style: .relative))
      - Top signatures (first 1-2, truncated)
      - "View Details" button -- navigates to DiagnosticsView with the saved diagnosis data (loads from log file ref and re-classifies, or shows cached summary)
      - "Re-analyze" button -- re-runs classification on the referenced log file
    - Footer: "Clear diagnostics history" button (destructive style, with confirmation alert)
    - Section: "Enhanced Logging" with WineDebugPreset picker
      - Picker showing presets: Normal (default), Crash, DLL/Load, Verbose
      - "Re-run with enhanced logging" button that sets the preset on ProgramSettings and triggers a new program launch
      - Label: Verbose preset shows "(generates large logs)" warning

    **WineDebugPreset picker:**
    Use a standard Picker with .segmented or .menu style showing all 4 presets.
    Each option shows: preset.displayName + preset.description (in secondary text).
    Binding to program.settings.activeWineDebugPreset.
    "Re-run" button is only enabled when a non-normal preset is selected.

    GPL v3 file header on all files.
