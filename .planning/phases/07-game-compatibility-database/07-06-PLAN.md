**GameConfigBannerView.swift:**
Create `GameConfigBannerView` struct:

**Properties:**
- `let matchResult: MatchResult`
- `@ObservedObject var bottle: Bottle`
- `let programURL: URL?`
- `@State private var isDismissed: Bool = false` -- "Not this" suppression
- `@State private var showDetail: Bool = false`

**Body:**
If not isDismissed, show a compact banner HStack:
- Leading: gamecontroller.fill icon in accent color
- Text: "Configuration available for \(matchResult.entry.title)" in `.callout`
- "Apply..." button that navigates to detail/preview
- "Why?" button that shows popover with matchResult.explanation
- "Not this" button (xmark icon) that sets isDismissed = true

Use `.background(Color.accentColor.opacity(0.08))`, `.clipShape(RoundedRectangle(cornerRadius: 8))`, `.padding(.horizontal)`.

The banner can be embedded in any view to show contextual suggestions.

**ProgramOverrideSettingsView.swift:**
Add game configuration suggestion section:

At the top of the view (before existing override groups), add a computed section:

```swift
@State private var gameMatch: MatchResult?
@State private var showGameConfigDetail: Bool = false
```

In `.onAppear` or `.task`:
1. Build `ProgramMetadata` from the program: exe name from program.url.lastPathComponent, exe URL, try to get file size and PE timestamp if available, try `SteamAppManifest.findAppIdForProgram(at: program.url)`.
2. Call `GameMatcher.bestMatch(metadata:, against: GameDBLoader.loadDefaults())`
3. Store in `gameMatch`

If gameMatch is non-nil, show a Section "Recommended Configuration":
- `GameConfigBannerView(matchResult: gameMatch, bottle: bottle, programURL: program.url)`
- Or a more detailed inline display: entry title, rating badge, "View & Apply..." NavigationLink to GameEntryDetailView

If gameMatch is nil, show nothing (no empty state in program settings).

Use `.sheet(isPresented: $showGameConfigDetail)` to present the detail view modally from program settings context.

If snapshot exists, add a Section in the config form (after existing sections, before repair/diagnostics):
- Section title: "Applied Game Configuration"

  - If revertedVerbs is non-empty, show additional info: "Settings reverted; installed components remain: \(verbs.joined(separator: ", "))"
