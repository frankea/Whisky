<task type="auto">
  <name>Task 1: Create ControllerMonitor with GCController integration</name>
  <files>
    Whisky/Utils/ControllerMonitor.swift
  </files>
  <action>
Create a new file `Whisky/Utils/ControllerMonitor.swift` with GPL v3 header. Add to the Whisky app target (not WhiskyKit -- GameController framework is app-level).

Define a public struct `ControllerInfo: Identifiable, Sendable` with:
- `id: UUID` (generated)
- `name: String` (from controller.vendorName ?? "Unknown Controller")
- `productCategory: String` (from controller.productCategory)
- `batteryLevel: Float?` (0.0...1.0, nil if not available)
- `batteryState: String?` (charging/discharging/full/unknown -- use String not GCDeviceBattery.State for Sendable)
- `isAttachedToDevice: Bool` (true = USB, false = Bluetooth/wireless)
- Computed `typeBadge: ControllerType` based on productCategory ("DualShock 4"/"DualSense" -> .playStation, "Xbox" contains -> .xbox, else .generic)
- Computed `connectionType: ConnectionType` based on isAttachedToDevice (.usb or .bluetooth)

Define `ControllerType: String, Sendable` enum with cases: .playStation, .xbox, .generic. Add `sfSymbol: String` computed property (all return "gamecontroller" for now). Add `displayName: String` property.

Define `ConnectionType: String, Sendable` enum with cases: .usb("USB"), .bluetooth("Bluetooth"). Add `sfSymbol: String` ("cable.connector" for USB, "wave.3.right" for Bluetooth).

Create `@MainActor class ControllerMonitor: ObservableObject` with:
- `@Published var controllers: [ControllerInfo] = []`
- `@Published var lastRefreshed: Date = Date()`
- Private notification observers stored for cleanup in deinit
- `func startMonitoring()`: Register for .GCControllerDidConnect and .GCControllerDidDisconnect notifications, then call refresh()
- `func stopMonitoring()`: Remove notification observers
- `func refresh()`: Query `GCController.controllers()`, map to ControllerInfo, update `controllers` and `lastRefreshed`
- `deinit` calls stopMonitoring()

Import GameController framework. Add `import os.log` with logger category "ControllerMonitor". Log controller count on refresh.

Handle the GCController pitfall: register for notifications before calling controllers() so late-discovered controllers update the list.

Add a lightweight `ControllerHistoryEntry: Codable, Sendable` struct with `name: String`, `connectionType: String`, `lastSeen: Date` for the diagnostics/export recent history requirement. Add `var recentHistory: [ControllerHistoryEntry]` on ControllerMonitor that appends on each refresh (bounded to last 10 entries, deduped by name).

`xcodebuild -project Whisky.xcodeproj -scheme Whisky -configuration Debug build` succeeds. `grep -r "ControllerMonitor" Whisky/` returns hits.
  </verify>

ControllerMonitor discovers connected controllers via GCController framework with type/connection/battery info. Connect/disconnect notifications update the list in real time. Recent controller history is tracked for diagnostics export.

4. Also ensure `disableControllerMapping` still maps to its separate SDL hint (these are different hints -- `USE_BUTTON_LABELS` controls physical vs logical mapping, `disableControllerMapping` was about the controller database).

This follows the existing pattern where each BottleInputConfig bool maps to an SDL env var in the inputCompatibility layer.
