    Create DLLOverrideEditor.swift -- a reusable SwiftUI view for editing DLL overrides:

    ```swift
    struct DLLOverrideEditor: View {
        // Managed overrides (read-only display)
        let managedOverrides: [(entry: DLLOverrideEntry, source: String)]
        // Custom overrides (editable)
        @Binding var customOverrides: [DLLOverrideEntry]
        // Warnings from DLLOverrideResolver
        let warnings: [DLLOverrideWarning]

        @State private var newDLLName: String = ""
        @State private var newDLLMode: DLLOverrideMode = .nativeThenBuiltin
    }
    ```

    The editor view layout:
    1. **Managed overrides section** (if non-empty): A List/ForEach showing each managed entry as a read-only row with:
       - Lock icon (Image(systemName: "lock.fill")) tinted secondary
       - DLL name text
       - Mode text (e.g., "Native, Builtin (n,b)")
       - Source label in lighter color (e.g., "Set by DXVK" or "Set by Steam fixes")

    2. **Custom overrides section**: A List/ForEach with:
       - Each row: DLL name (Text), Mode (Picker with DLLOverrideMode.allCases, using displayName)
       - Swipe-to-delete or minus button to remove entries
       - If a warning exists for this DLL, show a small warning icon (Image(systemName: "exclamationmark.triangle.fill")) with tooltip text from the warning message

    3. **Add row**: HStack at bottom with:
       - TextField for DLL name (placeholder: "dllname", no .dll suffix needed)
       - Picker for mode (DLLOverrideMode)
       - "+" Button to add entry (disabled if newDLLName is empty or already exists)
       - On add: append DLLOverrideEntry(dllName: newDLLName.lowercased(), mode: newDLLMode), clear newDLLName

    4. **Presets button**: A Menu button titled "Presets..." with items:
       - "DXVK (recommended)" -- applies DLLOverrideResolver.dxvkPreset entries: for each preset entry, if not already in customOverrides, append it; if already present, update its mode
       - Future presets can be added here

    5. **Optional "Advanced" toggle**: A DisclosureGroup labeled "Advanced" containing a raw WINEDLLOVERRIDES text field. This is an escape hatch; edits to the raw string are NOT parsed back into structured entries (read-only display of the rendered string). Show the resolved string from DLLOverrideResolver for reference. Per Claude's discretion, this can be deferred if it adds complexity without clear value in this phase.

    Create DLLOverrideConfigSection.swift -- the bottle-level section for ConfigView:

    ```swift
    struct DLLOverrideConfigSection: View {
        @ObservedObject var bottle: Bottle
        @Binding var isExpanded: Bool
    }
    ```

    The section:
    - Uses `Section("config.title.dllOverrides", isExpanded: $isExpanded)` pattern (matching DXVKConfigSection)
    - Computes managed overrides from bottle state: if bottle.settings.dxvk, add DXVK managed entries with source "DXVK"; if launcher has requiresDXVK and autoEnableDXVK, add launcher entries
    - Uses DLLOverrideEditor with `managedOverrides` computed above and `customOverrides` bound to `$bottle.settings.dllOverrides`
    - Computes warnings using DLLOverrideResolver

    Update ConfigView.swift:
    - Add `@AppStorage("dllOverrideSectionExpanded") private var dllOverrideSectionExpanded: Bool = false`
    - Add `DLLOverrideConfigSection(bottle: bottle, isExpanded: $dllOverrideSectionExpanded)` after the Performance section and before Stability
    - Add `.animation(.whiskyDefault, value: dllOverrideSectionExpanded)`

    Localization keys to add:
    - "config.title.dllOverrides" = "DLL Overrides"
    - "config.dllOverrides.managed" = "Managed"
    - "config.dllOverrides.custom" = "Custom"
    - "config.dllOverrides.addDLL" = "Add DLL Override"
    - "config.dllOverrides.presets" = "Presets..."
    - "config.dllOverrides.preset.dxvk" = "DXVK (recommended)"
    - "config.dllOverrides.source.dxvk" = "Set by DXVK"
    - "config.dllOverrides.source.launcher" = "Set by Launcher fixes"

    Create ProgramOverrideSettingsView.swift -- the per-program override UI:

    ```swift
    struct ProgramOverrideSettingsView: View {
        @ObservedObject var bottle: Bottle
        let program: Program

        // Track which groups are in "Override" mode locally
        // (derived from whether the corresponding ProgramOverrides fields are non-nil)
    }
    ```

    Override groups (per locked decision):

    1. **Graphics / DXVK group**:
       - Toggle: "Override Graphics Settings" (on if program.settings.overrides?.dxvk != nil)
       - When off (inherit): Show "Currently inherited: DXVK \(bottle.settings.dxvk ? "On" : "Off"), Async \(bottle.settings.dxvkAsync ? "On" : "Off"), HUD \(bottle.settings.dxvkHud)"
       - When on (override): Show Toggle for DXVK, Toggle for Async, Picker for HUD
       - Copy-on-enable: When toggling from inherit to override, initialize overrides with current bottle values: `program.settings.overrides = ProgramOverrides(dxvk: bottle.settings.dxvk, dxvkAsync: bottle.settings.dxvkAsync, dxvkHud: bottle.settings.dxvkHud)`
       - When toggling from override to inherit: set dxvk, dxvkAsync, dxvkHud fields to nil

    2. **Sync group**:
       - Toggle: "Override Sync Settings"
       - Inherit summary: "Currently inherited: \(bottle.settings.enhancedSync)"
       - Override: Picker for EnhancedSync
       - Copy-on-enable: enhancedSync = bottle.settings.enhancedSync

    3. **Performance group**:
       - Toggle: "Override Performance Settings"
       - Inherit summary: "Currently inherited: \(bottle.settings.performancePreset), Shader Cache \(bottle.settings.shaderCacheEnabled ? "On" : "Off"), Force D3D11 \(bottle.settings.forceD3D11 ? "On" : "Off")"
       - Override: Picker for PerformancePreset, Toggle for ShaderCache, Toggle for ForceD3D11
       - Copy-on-enable: performancePreset, shaderCacheEnabled, forceD3D11

    4. **Input group**:
       - Toggle: "Override Input Settings"
       - Inherit summary: "Currently inherited: Controller Compat \(bottle.settings.controllerCompatibilityMode ? "On" : "Off")"
       - Override: Toggle for controllerCompatibilityMode, disableHIDAPI, allowBackgroundEvents, disableControllerMapping
       - Copy-on-enable: all 4 input fields

    5. **DLL Overrides group**:
       - Toggle: "Override DLL Overrides"
       - Inherit summary: "Currently inherited: \(bottle.settings.dllOverrides.count) custom overrides"
       - Override: DLLOverrideEditor with managedOverrides same as bottle level, customOverrides bound to program.settings.overrides.dllOverrides
       - Copy-on-enable: dllOverrides = bottle.settings.dllOverrides (copy current bottle custom overrides as starting point)

    6. **Reset Overrides button** at the bottom:
       - "Reset All Overrides" button (destructive style)
       - Sets program.settings.overrides to nil (or ProgramOverrides() with all nils)
       - Confirmation alert: "Reset all per-program overrides? This program will inherit all settings from the bottle."

    Layout pattern for each group: Use `Section` with the group title. Inside:
    - `Toggle("Override [Group] Settings", isOn: overrideBinding)` where overrideBinding is a computed Binding that checks if fields are nil
    - When override is on, show the controls indented or in a sub-section
    - When override is off, show the "Currently inherited: ..." text in secondary color

    Helper: Create a private helper method or Binding extension for the inherit/override toggle pattern to avoid duplicating the nil-check + copy-on-enable logic for each group.

    Update ProgramView.swift:
