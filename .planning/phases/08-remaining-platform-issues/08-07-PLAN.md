  <action>
Create `Whisky/Utils/SteamDownloadMonitor.swift` with GPL v3 header:

1. Define `StallStatus: Sendable` enum: `.noDownloads`, `.downloading`, `.likelyStalled(duration: TimeInterval)`, `.confirmedStall(duration: TimeInterval, evidence: [String])`.

2. Define `@MainActor class SteamDownloadMonitor: ObservableObject`:
   - `@Published var status: StallStatus = .noDownloads`
   - `@Published var isMonitoring: Bool = false`
   - Private: `lastSnapshot: [String: (size: UInt64, mtime: Date)] = [:]`
   - Private: `stallStartTime: Date?`
   - Private: `stallThreshold: TimeInterval = 180` (3 minutes, per user decision)
   - Private: `samplingInterval: TimeInterval = 45` (30-60s range, per Claude's discretion)
   - Private: `monitoringTask: Task<Void, Never>?`
   - Private: `alertedThisSession: Set<String> = []` (per-bottle rate limiting, per user decision)

3. `func startMonitoring(bottleURL: URL)`:
   - Gate: only start if launcher is Steam (check bottle.settings.detectedLauncher == .steam)
   - Create a Task that loops: sleep for samplingInterval, call sample(bottleURL:), update status
   - Set isMonitoring = true

4. `func stopMonitoring()`: Cancel the task, set isMonitoring = false, reset state.

5. `func sample(bottleURL: URL)`:
   - Check `drive_c/Program Files (x86)/Steam/steamapps/downloading/` for subdirs
   - Build snapshot of subdirectory names + total sizes + newest mtime
   - Compare with lastSnapshot: if no progress for >= stallThreshold, status = .likelyStalled
   - Optional corroboration: check if Wine logs contain WinHTTP/WinINet timeout patterns for higher confidence (.confirmedStall)

6. When status transitions to .likelyStalled or .confirmedStall:
   - Post a `Notification.Name.steamDownloadStallDetected` with userInfo containing the stall evidence
   - Only post once per bottle per session (check alertedThisSession)
   - Per user decision: rate-limited, once per bottle per session with "Don't warn again" choice

7. Add logger with category "SteamDownloadMonitor".

In patterns.json, add a new pattern entry:
```json
{
  "id": "steam-download-stall",
  "category": "networkingLaunchers",
  "severity": "warning",
  "confidence": 0.6,
  "substringPrefilter": "download",
  "regex": "(?:Steam|steam|content_log).*(?:download|HTTP).*(?:stall|timeout|failed|timed out|connection reset)",
  "tags": ["steam", "download", "stall", "network"],
  "captureGroups": null,
  "remediationActionIds": ["steam-download-fix"]
}
```

In remediations.json, add:
```json
{
  "id": "steam-download-fix",
  "title": "Fix Steam Download Stall",
  "description": "Steam downloads can stall due to network configuration. Try increasing the network timeout or switching download region.",
  "category": "networkingLaunchers",
  "actionType": "changeSetting",
  "risk": "low",
  "settingKeyPath": "networkTimeout",
  "settingValue": "120000",
  "winetricksVerb": null,
  "whatWillChange": "Increases network timeout and enables HTTP/1.1 fallback for Steam downloads",
  "undoPath": "Reset network timeout in Launcher settings",
  "appliesNextLaunch": true
}
```

Bump the version number in both JSON files if they have one.
