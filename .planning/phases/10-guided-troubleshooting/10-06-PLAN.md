    **TroubleshootingWizardView.swift**: Main wizard sheet presented as a .sheet. Structure:
    - Accepts: `bottle: Bottle`, `program: Program?`, `entryContext: EntryContext`.
    - Creates: `@StateObject var engine: TroubleshootingFlowEngine` initialized with FlowLoader.loadAllFlows(), FlowLoader.loadFragments(), CheckRegistry(), TroubleshootingSessionStore().
    - On appear: Check for active session via sessionStore. If found, show SessionResumeView overlay. If entryContext has initial evidence, populate preflight accordingly. Collect preflight data via PreflightCollector.
    - Layout: HStack with ProgressRailView on the left (fixed width ~180pt) and scrollable step area on the right.
    - Step area: Shows current step card based on engine.currentNode. Switch on session.phase:
      - .symptom -> SymptomPickerView
      - .checks -> StepCardView for check nodes (with progress indicator when isRunningCheck)
      - .fix -> FixPreviewView
      - .verify -> FixVerifyView
      - .export -> resolved summary or export options
      - .escalation -> EscalationView
    - Toolbar: "Close" button (left), step number "Step X of Y" (center), "Back" and "Skip" buttons (right). Per locked decision: step numbering relative to current path.
    - Show BranchExplanationView inline when engine.pathChanged is true.
    - Deep links: Config and Logs buttons open as sub-sheets (.sheet) that return to the same wizard step per locked decision.
    - Minimum window size: ~700x500 to accommodate rail + step area.

    **ProgressRailView.swift**: Vertical progress rail on the left side. Per locked decision: 5 stable phases with state-based styling.
    - Define `RailPhase` struct: phase (SessionPhase), title (String), sfSymbol (String).
    - 5 fixed phases: Symptom (questionmark.circle), Checks (magnifyingglass.circle), Fix (wrench.and.screwdriver), Verify (checkmark.circle), Export (square.and.arrow.up.circle).
    - For each phase: Show icon + title. Styling based on session state:
      - Completed: green checkmark, bold title
      - Active: accent color, filled icon
      - Upcoming: gray, unfilled icon
      - Superseded (future step changed by branching): dimmed with subtle strikethrough style per locked decision
    - Connector lines between phases (vertical 2pt rectangles) colored by completion state.
    - Branch-specific sub-steps appear under the active phase as indented items per locked decision.

    **SymptomPickerView.swift**: Grid/list of 8 symptom categories. Each category shown as a card/button with:
    - SF Symbol (from SymptomCategory.sfSymbol)
    - Title (from SymptomCategory.displayTitle)
    - Brief description
    - On tap: call engine.selectCategory(category)
    - Layout: 2-column grid for wider views, single column for narrow. Use LazyVGrid with adaptive columns.
    - "Other" option shown at bottom with less visual weight per locked decision.

    All views: GPL v3 header with target "Whisky". Follow existing SwiftUI patterns.

  <action>
    **StepCardView.swift**: Renders a single flow step node as a card. Accepts: `node: FlowStepNode`, `checkResult: CheckResult?`, `isRunning: Bool`. Layout:
    - Card container with rounded rectangle background, padding, shadow.
    - Title (node.title) in headline font.
    - Description (node.description) in subheadline, secondary color.
    - For check nodes: Show evidence from checkResult using evidenceMap to display labels. Show confidence badge using ConfidenceTier color scheme (green/yellow/gray, matching Phase 5 pattern). Show ProgressView when isRunning.
    - For info nodes: Show description with relevant icon.
    - For branch nodes: Show "Why path changed" if applicable.
    - Step number shown in a circle badge at top-left of card.

    **FixPreviewView.swift**: Shown when engine reaches a fix node. Accepts: `node: FlowStepNode`, `engine: TroubleshootingFlowEngine`, `bottle: Bottle`, `program: Program?`. Layout:
    - Title: node.title (e.g., "Switch to DXVK")
    - Description: node.description
    - Diff-style preview section: Show "Current" and "New" values side by side. Use FixApplicator.preview() to get current/new values. Style: monospace font, red for current (being removed), green for new (being applied). Per locked decision: diff-style preview.
    - Reversibility indicator: If node.isReversible == true, show "This change can be undone". If false, show "This action cannot be undone" in warning style. Per locked decision: clearly label non-reversible.
    - "Apply Fix" button (prominent, accent color). Per locked decision: explicit gated apply.
    - If node.requiresConfirmation == true, show a confirmation alert before applying. Per locked decision: high-impact changes get additional confirmation.
    - On apply: Call FixApplicator.apply(), then engine.applyFix(). Transition to verify phase.
    - "Skip for now" button (secondary style).

    **FixVerifyView.swift**: Shown after a fix is applied. Accepts: `engine: TroubleshootingFlowEngine`. Layout:
    - Title: "Did this fix the problem?"
    - Description: "Try running your program again to check if the issue is resolved."
    - Two prominent buttons: "Yes, it's fixed" (green/success style) and "No, still broken" (secondary style).
    - "Yes" calls engine.userReportsFixed(). "No" calls engine.userReportsNotFixed().
    - Show count of remaining fix attempts: "Fix attempt X of 3" per locked 3-attempt cap.
    - If engine shows undo is available for last fix, include "Undo last fix" button.

    All views: GPL v3 header, target "Whisky".

    **SessionResumeView.swift**: Overlay shown when a paused session is found. Per locked session persistence decisions. Accepts: `session: TroubleshootingSession`, `stalenessChanges: [StalenessChange]`, `onResume: () -> Void`, `onStartOver: () -> Void`, `onDiscard: () -> Void`. Layout:

    - Summary: "You have a paused session for [symptomCategory displayTitle]"

    - Per locked decision: explicit Resume/Start over/Discard actions.

    **EscalationView.swift**: Shown when the flow reaches escalation (unresolved after 3 fix attempts or other dead end). Per locked escalation path decisions. Accepts: `engine: TroubleshootingFlowEngine`, `bottle: Bottle`, `program: Program?`. Layout:
