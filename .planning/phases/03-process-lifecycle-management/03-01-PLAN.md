<task type="auto">
  <name>Task 1: Create WineProcess shared type definitions</name>
  <files>WhiskyKit/Sources/WhiskyKit/Wine/WineProcessTypes.swift</files>
  <action>
Create a new file `WhiskyKit/Sources/WhiskyKit/Wine/WineProcessTypes.swift` with GPL v3 header for the WhiskyKit target.

Define the following public types:

**ProcessKind** enum (String, CaseIterable, Codable, Sendable):
- Cases: `.app`, `.service`, `.system`
- Static `knownServiceProcesses: Set<String>` containing lowercase names: "services.exe", "winedevice.exe", "plugplay.exe", "svchost.exe", "rpcss.exe", "tabtip.exe", "conhost.exe"
- Static `knownSystemProcesses: Set<String>` containing lowercase names: "explorer.exe", "start.exe", "csrss.exe", "wininit.exe", "winlogon.exe", "lsass.exe", "smss.exe"
- Static `func classify(_ imageName: String) -> ProcessKind` that lowercases the input and checks against known sets, defaulting to `.app`

**ProcessSource** enum (String, Codable, Sendable):
- Cases: `.whisky` (launched by Whisky and tracked in ProcessRegistry), `.untracked` (discovered via tasklist but not in registry)

**ShutdownState** enum (String, Sendable):
- Cases: `.idle`, `.stopping`, `.forceKilling`

**WineProcess** struct (Identifiable, Hashable, Sendable):
- `imageName: String` (Windows image name from tasklist, e.g. "game.exe")
- `winePID: Int32` (Wine/Windows PID from tasklist)
- `memoryUsage: String` (raw string from tasklist, e.g. "24 K")
- `kind: ProcessKind`
- `source: ProcessSource`
- `launchTime: Date?` (from ProcessRegistry match, nil for untracked)
- `macosPID: Int32?` (from ProcessRegistry match, nil for untracked)
- `commandLine: String?` (for detail drawer, nil if unknown)
- Identifiable via `id: Int32 { winePID }`
- Hashable via winePID + imageName

Ensure all types have `public` access for cross-module use. Use `import Foundation` only (no AppKit/SwiftUI).
  </action>

Run `swift build --package-path WhiskyKit` to confirm compilation. Verify the file exists and contains all four types.
  </verify>
  <done>WineProcessTypes.swift compiles, exports WineProcess, ProcessKind, ProcessSource, and ShutdownState. ProcessKind.classify("services.exe") returns .service, ProcessKind.classify("game.exe") returns .app.</done>

**Wine+ProcessManagement.swift** (NEW file with GPL v3 header for WhiskyKit target):

Create a `public extension Wine` with these static methods:

1. **`isWineserverRunning(for bottle: Bottle) async -> Bool`** (@MainActor):
   - Run `wineserver -k0` via `Wine.runWineserverProcess(name: "wineserver-probe", args: ["-k0"], bottle: bottle)`.
   - Iterate the AsyncStream looking for `.terminated(let code)` output.
   - Return `true` if exit code is 0 (wineserver running), `false` otherwise.
   - Wrap in do/catch, return `false` on any error.
   - Log at debug level: "Wineserver probe for '\(bottle.settings.name)': \(running ? "active" : "idle")"

2. **`parseTasklistOutput(_ output: String) -> [WineProcess]`** (not @MainActor, pure function):
   - Split output on newlines (omitting empty subsequences).
   - For each line, split on "," separator.
   - Strip surrounding `"` quotes and whitespace from each field.
   - Skip header line (where field[1] == "PID").
   - Require at least 5 fields: Image Name, PID, Session Name, Session#, Mem Usage.
   - Parse fields[1] as Int32 for winePID (skip line if parse fails).
   - Use `ProcessKind.classify(fields[0])` for kind.
   - Default source to `.untracked` (caller will update after merge).
   - Return `[WineProcess]`.

3. **`gracefulKillProcess(winePID: Int32, bottle: Bottle) async`** (@MainActor):
   - Run `taskkill.exe /PID \(winePID)` via `Wine.runWine(["taskkill.exe", "/PID", String(winePID)], bottle: bottle)`.
   - Log result. Ignore errors (process may have already exited).

4. **`forceKillProcess(winePID: Int32, bottle: Bottle) async`** (@MainActor):
   - Run `taskkill.exe /PID \(winePID) /F` via `Wine.runWine(["taskkill.exe", "/PID", String(winePID), "/F"], bottle: bottle)`.
   - Log result. Ignore errors.

Use the existing Logger pattern: `private static let processLogger = Logger(subsystem: Bundle.whiskyBundleIdentifier, category: "Wine.ProcessManagement")`

**ProcessRegistry.swift** enhancements:

1. Add `public func getProcessCount(for bottle: Bottle) -> Int`:
   - Lock, read `activeProcesses[bottle.url]?.count ?? 0`, unlock, return.

2. Add `public func getProcessCount(for bottleURL: URL) -> Int`:
   - Same as above but takes URL directly (for use from non-@MainActor contexts where Bottle reference is unavailable).

3. Make `clearRegistry(for:)` public (change from `private` to `public`) so the ViewModel can clear after shutdown.

4. Add `public func hasActiveProcesses(for bottleURL: URL) -> Bool`:
   - Returns `getProcessCount(for: bottleURL) > 0`.
  </action>
