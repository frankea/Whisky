**ProcessesViewModel** (@MainActor, ObservableObject):

Properties:
- `@Published var processes: [WineProcess] = []` -- merged, sorted process list
- `@Published var isPolling: Bool = false`
- `@Published var shutdownState: ShutdownState = .idle`
- `@Published var filterMode: FilterMode = .appsOnly`
- `@Published var selectedProcessID: Int32?` -- selected Wine PID for table selection
- `private var pollingTask: Task<Void, Never>?`
- `private let bottle: Bottle`
- `private let pollingInterval: TimeInterval = 3.0` -- tunable, ~3s per user decision

**FilterMode** enum (nested, CaseIterable):
- `.appsOnly` -- default, shows only ProcessKind.app
- `.all` -- shows all processes including services and system

**Computed properties:**
- `var filteredProcesses: [WineProcess]` -- applies filterMode to processes array
- `var appCount: Int` -- count of .app processes (for running indicator)

**Initializer:**
- `init(bottle: Bottle)` -- stores bottle reference

**Polling lifecycle:**
- `func startPolling()`:
  - Guard pollingTask == nil to prevent multiple concurrent polls.
  - Set isPolling = true.
  - Create pollingTask as Task that loops: await refreshProcessList(), then try? await Task.sleep(for: .seconds(pollingInterval)), checking Task.isCancelled.
- `func stopPolling()`:
  - Cancel pollingTask, set to nil, set isPolling = false.

**Data refresh:**
- `func refreshProcessList()`:
  - Guard shutdownState == .idle (skip refresh during shutdown to avoid race condition per research pitfall 3).
  - Run `tasklist.exe` via `try await Wine.runWine(["tasklist.exe", "/FO", "CSV"], bottle: bottle)` to get CSV output.
  - Parse with `Wine.parseTasklistOutput(output)`.
  - Get registry processes via `ProcessRegistry.shared.getProcesses(for: bottle)`.
  - Merge: for each tasklist process, check if a registry entry matches by program name (case-insensitive imageName comparison). If matched, set source = .whisky, populate launchTime and macosPID from registry info. Unmatched tasklist entries stay as .untracked.
  - Also: check for stale registry entries (in registry but not in tasklist) and unregister them to prevent ghost entries (per research pitfall 6).
  - Sort by kind (.app first, then .service, then .system), then by imageName alphabetically.
  - Assign to self.processes on MainActor.
  - Handle errors gracefully: if tasklist fails, log and keep previous process list.

**Kill actions:**
- `func quitProcess(_ process: WineProcess)`:
  - Call `await Wine.gracefulKillProcess(winePID: process.winePID, bottle: bottle)`.
  - Wait 1 second, then refresh.
- `func forceQuitProcess(_ process: WineProcess)`:
  - Call `await Wine.forceKillProcess(winePID: process.winePID, bottle: bottle)`.
  - Wait 0.5 seconds, then refresh.

**Shutdown orchestration (3-step per user decision):**
- `func stopBottle() async -> Int`:
  - Set shutdownState = .stopping.
  - Stop polling.
  - Step 1: Graceful per-app kill -- filter processes for .app kind, call `Wine.gracefulKillProcess` for each. Wait 4 seconds (internal constant).
  - Refresh process list (temporarily allow refresh during shutdown for this check).
  - Step 2: If processes remain, set shutdownState = .forceKilling. Call `Wine.killBottle(bottle: bottle)` (wineserver -k). Wait 2 seconds.
  - Step 3: SIGKILL remaining tracked macOS PIDs from ProcessRegistry as last resort. For each remaining registry entry with pid > 0, call `kill(pid, SIGKILL)`.
  - Clear registry via `ProcessRegistry.shared.clearRegistry(for: bottle.url)`.
  - Set shutdownState = .idle.
  - Refresh and restart polling.
  - Return count of processes that were stopped.
- `func forceStopBottle() async -> Int`:
  - Set shutdownState = .forceKilling.
  - Stop polling.
  - Skip step 1, go directly to wineserver -k + SIGKILL fallback.
  - Clear registry, set idle, refresh.
  - Return count.

Add `import WhiskyKit`, `import SwiftUI`, `import os.log`. Use logger: `private let logger = Logger(subsystem: Bundle.whiskyBundleIdentifier, category: "ProcessesViewModel")`.
