  <name>Task 1: RunLogEntry and RunLogHistory data models with persistence</name>
  <files>
    WhiskyKit/Sources/WhiskyKit/Whisky/RunLog.swift

  <action>
1. **Create RunLog.swift** with:

   a. **RunLogEntry** struct (public, Codable, Identifiable, Equatable, Sendable):
      - `public let id: UUID`
      - `public let startTime: Date`
      - `public var endTime: Date?`
      - `public var exitCode: Int32?`
      - `public let logFileName: String` (relative to logs folder -- not absolute URL, so it survives bottle moves)
      - `public let programName: String` (display name for list)
      - `public var activeWineDebugPreset: String?` (name of debug preset if diagnostic run)
      - `public var hasWineDebugOutput: Bool = false`
      - Computed: `public var isRunning: Bool { endTime == nil }`
      - Computed: `public var duration: TimeInterval? { guard let end = endTime else { return nil }; return end.timeIntervalSince(startTime) }`
      - Init: `public init(programName: String, logFileName: String)` sets id = UUID(), startTime = Date()

   b. **RunLogHistory** struct (public, Codable, Equatable):
      - `public static let maxEntriesPerProgram = 10`
      - `public var entries: [RunLogEntry] = []`
      - `public mutating func append(_ entry: RunLogEntry)` -- adds entry, prunes oldest if count > max, returns removed entries for caller to delete log files
      - `public mutating func markCompleted(id: UUID, exitCode: Int32, hasWineDebug: Bool)` -- finds entry by id, sets endTime and exitCode
      - `public func logFileURLs(in logsFolder: URL) -> [URL]` -- resolves all logFileName entries to full URLs
      - `public mutating func deleteEntry(id: UUID) -> RunLogEntry?` -- removes specific entry, returns it for log file cleanup
      - `public mutating func deleteOldEntries(keepLast: Int = maxEntriesPerProgram) -> [RunLogEntry]` -- explicit pruning returning removed entries

   c. **RunLogStore** caseless enum (public) for persistence operations:
      - `static func historyURL(for programName: String, in bottleURL: URL) -> URL` -- returns `{bottle}/Program Settings/{programName}.run-history.plist` (separate file from main settings per research recommendation)
      - `static func load(for programName: String, in bottleURL: URL) -> RunLogHistory` -- decode from plist, return empty on failure
      - `static func save(_ history: RunLogHistory, for programName: String, in bottleURL: URL)` -- encode to plist
      - `static func totalLogSize(in logsFolder: URL) -> Int64` -- sum of all .log files for size cap enforcement

2. **ProgramSettings.swift:** No changes needed -- run history is stored in separate `.run-history.plist` file to avoid bloating the main settings file.

3. **RunLogTests.swift:** Write unit tests:
   - Test entry creation with auto-generated id and startTime
   - Test append + auto-prune at 10 entries (verify oldest removed)
   - Test markCompleted updates endTime and exitCode
   - Test deleteEntry returns removed entry
   - Test isRunning computed property
   - Test duration computed property
   - Test Codable round-trip for RunLogEntry and RunLogHistory
  </action>
  <verify>`swift test --package-path WhiskyKit --filter RunLogTests` passes all tests.</verify>

<task type="auto">
  <name>Task 2: Integrate RunLog with Wine.runProgram process execution</name>
  <files>
    WhiskyKit/Sources/WhiskyKit/Wine/Wine.swift
  </files>
  <action>
1. **Wine.runProgram integration:** In the main `runProgram(at:args:bottle:environment:)` method:
   - At the start of execution (after creating the file handle/log URL):
     - Create a `RunLogEntry(programName: url.lastPathComponent, logFileName: logFileURL.lastPathComponent)`
     - If a WineDebugPreset is active in the environment (check for WINEDEBUG key), set `entry.activeWineDebugPreset` to the preset name
     - Load the program's RunLogHistory via `RunLogStore.load(for:in:)` using bottle URL
     - Append the new entry, save immediately so the "running" state is persisted
   - After the process exits (where exitCode is captured):
     - Reload history, call `markCompleted(id: entry.id, exitCode: exitCode, hasWineDebug: environment.keys.contains("WINEDEBUG"))`
     - Save updated history
   - The existing log file creation (`makeFileHandleWithURL`) already writes to disk. The RunLogEntry just tracks metadata about it.

2. **Dual streaming consideration:** The existing code already writes process output to a log file via FileHandle. The UI (ProcessOutput AsyncStream) also receives the output. So dual streaming (UI + disk) is already in place. No changes needed for the streaming itself.

3. **ProgramRunResult extension:** Add `runLogEntryId: UUID` field to `ProgramRunResult` so callers can correlate the run result with the log entry. This is useful for the console UI (Plan 06) to open the correct log entry after a run.

4. **Avoid breaking existing callers:** The `runProgram` method is @discardableResult, so adding the runLogEntryId to ProgramRunResult is backward-compatible. Existing callers that discard the result are unaffected.

5. **Log file naming:** The existing log file naming uses date strings. Verify that log filenames are unique enough to prevent collisions (they use ISO 8601 with fractional seconds -- should be fine). If not unique enough, append a short UUID suffix.
  </action>
