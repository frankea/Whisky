  **AppDelegate lifecycle hooks:**

  In `applicationDidFinishLaunching`:
  1. After the existing move-to-Applications alert logic, add startup cleanup in a background task:
     ```swift
     Task.detached {
         // Clean up orphaned temp files from previous sessions (older than 24 hours)
         await TempFileTracker.shared.cleanupOldFiles(olderThan: 24 * 60 * 60)

         // Zombie process sweep: graceful SIGTERM -> SIGKILL escalation for orphaned Wine processes
         // Only acts on processes matching known WINEPREFIX bottle paths
         let killedCount = await ProcessRegistry.shared.sweepOrphanedProcesses(bottles: BottleVM.shared.bottles)
         if killedCount > 0 {
             await MainActor.run {
                 // Show non-blocking toast per locked decision: "Cleaned up leftover Wine processes"
                 // Toast is shown via NotificationCenter post or a shared toast publisher
                 NotificationCenter.default.post(
                     name: .zombieProcessesCleaned,
                     object: nil,
                     userInfo: ["count": killedCount]
                 )
             }
         }
     }
     ```
     This catches temp files AND orphaned Wine processes from crashes. It runs asynchronously and never blocks launch.

  **Note on zombie process triggers:** The locked decision specifies 4 zombie process cleanup triggers: (1) app launch sweep, (2) bottle open check, (3) pre-launch reconciliation, (4) quit cleanup. This plan implements triggers (1) and (4). Triggers (2) bottle-open quick check and (3) pre-launch reconciliation require deeper integration with Wine process lifecycle tracking (matching WINEPREFIX, tracking wineserver PIDs per bottle) which is deferred to Phase 3: Process Lifecycle Management. Phase 1 provides the foundational cleanup-on-launch and cleanup-on-quit triggers.

  In `applicationWillTerminate`:
  1. The existing code checks `UserDefaults.standard.bool(forKey: "killOnTerminate")` and calls `WhiskyApp.killBottles()`. Enhance this to respect per-bottle killOnQuit overrides:
     ```swift
     func applicationWillTerminate(_ notification: Notification) {
         let globalKill = UserDefaults.standard.bool(forKey: "killOnTerminate")

         // Synchronous cleanup -- must not block termination
         var totalKilled = 0
         for bottle in BottleVM.shared.bottles {
             let bottlePolicy = bottle.settings.killOnQuit
             let shouldKill: Bool
             switch bottlePolicy {
             case .inherit:
                 shouldKill = globalKill
             case .alwaysKill:
                 shouldKill = true
             case .neverKill:
                 shouldKill = false
             }

             if shouldKill {
                 Wine.killBottle(bottle: bottle)
                 totalKilled += 1
             }
         }
         // Note: No toast on quit -- termination cannot be blocked per locked decision.
         // Force-kill toast is only relevant during app-launch sweep (above).

         // Best-effort synchronous temp file cleanup
         // TempFileTracker.cleanupAll is async, but we cannot await in applicationWillTerminate.
         // Use a synchronous approach: iterate tracked files and try to delete them.
         let trackedFiles = TempFileTracker.shared.getAllTrackedFiles()
         for (fileURL, _) in trackedFiles {
             try? FileManager.default.removeItem(at: fileURL)
         }
     }
     ```

     **Important:** `applicationWillTerminate` cannot use async/await. All cleanup must be synchronous. The TempFileTracker cleanup uses the synchronous `FileManager.removeItem` directly. ProcessRegistry cleanup via `Wine.killBottle` is already synchronous (it calls wineserver --kill synchronously).
