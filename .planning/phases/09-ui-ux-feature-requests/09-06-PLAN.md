<task type="auto">
  <name>Task 2: ConsoleLogView with channel filtering and export</name>
  <files>
    Whisky/Views/Programs/ConsoleLogView.swift
  </files>
  <action>
1. **Create ConsoleLogView.swift:**
   - Props: `let runEntry: RunLogEntry`, `let bottle: Bottle`
   - **Header:** Program name, start time, duration (or "Running..."), exit code
   - **Footer when completed:** "Exited (code X)" styled text (per user decision)

2. **Log content loading:**
   - Resolve log file: `Wine.logsFolder.appending(path: runEntry.logFileName)`
   - Read file contents. For large files, load lazily or paginate.
   - Parse lines into channels:
     - Lines starting with common WINEDEBUG patterns (e.g., `0009:`, `wine:`, `fixme:`, `err:`, `trace:`) -> classify as "winedebug" channel
     - Lines written to stderr (if distinguishable -- the current Wine logging may not distinguish). If the log format doesn't separate stdout/stderr, use a simpler two-channel model: "output" (all non-WINEDEBUG lines) and "winedebug" (WINEDEBUG lines).
     - Per user decision: "UI separates channels (stdout, stderr, winedebug) with filter/toggle controls"
   - If WINEDEBUG was not active for this run (check `runEntry.hasWineDebugOutput`), hide the winedebug filter toggle.

3. **Channel filter controls:**
   - Toolbar or top bar with toggle buttons: "stdout", "stderr", "WINEDEBUG"
   - Each toggle shows/hides lines from that channel
   - Default: stdout + stderr visible, WINEDEBUG hidden (if present)

4. **Log text display:**
   - Use monospaced font (`.system(.body, design: .monospaced)`)
   - ScrollView with lazy loading for performance
   - Syntax-like coloring: stderr lines in red, WINEDEBUG in gray, stdout in default
   - Auto-scroll to bottom for live (running) logs

5. **Actions:**
   - "Copy" -- copies visible (filtered) log content to clipboard
   - "Export..." -- save filtered or full log to file via NSSavePanel
   - "Open Logs Folder" -- `NSWorkspace.shared.open(Wine.logsFolder)`
   - If `runEntry.hasWineDebugOutput`, export includes WINEDEBUG content labeled clearly

6. **Live streaming for running processes:**
   - If `runEntry.isRunning`, poll the log file for new content (tail -f equivalent) using a Timer or `DispatchSource.makeFileSystemObjectSource` for file change monitoring
   - Append new lines to the view as they arrive
   - When process exits (detected by re-checking RunLogStore), update the footer to "Exited (code X)"

7. Add localization keys: `console.log.copy`, `console.log.export`, `console.log.filter.stdout`, `console.log.filter.stderr`, `console.log.filter.winedebug`, `console.log.liveIndicator`.
  </action>
  <verify>`xcodebuild -project Whisky.xcodeproj -scheme Whisky -configuration Debug build` succeeds.</verify>
