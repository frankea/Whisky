<task type="auto">
  <name>Task 2: FixApplicator with preview, apply, and undo</name>
  <files>
    WhiskyKit/Sources/WhiskyKit/Troubleshooting/FixApplicator.swift
  </files>
  <action>
    Define `FixApplicator` as a caseless enum (following GPUDetection/GameConfigApplicator pattern).

    **Static methods:**

    - `@MainActor static func preview(fixId: String, params: [String: String], bottle: Bottle, program: Program?) -> FixPreview?` -- Returns a preview of what the fix will change without applying it. FixPreview struct: `settingName: String`, `currentValue: String`, `newValue: String`, `scope: String` ("bottle" or "program"), `isReversible: Bool`. Switch on fixId to determine what setting to read and what the new value would be. Known fixIds:
      - "switch-backend" -> changes graphicsBackend
      - "enable-dxvk-async" -> enables DXVK async
      - "set-audio-driver" -> changes audio driver registry
      - "set-buffer-size" -> changes DirectSound buffer
      - "enable-esync" -> enables enhanced sync
      - "enable-controller-compat" -> enables controller compatibility mode
      - "install-winetricks-verb" -> non-reversible winetricks install
      - "install-dependency" -> non-reversible dependency install
      - "run-enhanced-diagnostics" -> applies WINEDEBUG preset
      - "restart-wineserver" -> non-reversible process restart
      - Unknown fixId -> return nil

    - `@MainActor static func apply(fixId: String, params: [String: String], bottle: Bottle, program: Program?) -> FixAttempt` -- Applies the fix. Creates FixAttempt with beforeValue captured from current state. Switch on fixId to apply:
      - "switch-backend" -> set bottle.settings.graphicsBackend (or program override)
      - "enable-dxvk-async" -> set bottle.settings DXVK async = true
      - "set-audio-driver" -> write audio driver to Wine registry
      - "set-buffer-size" -> write DirectSound buffer registry key
      - "enable-esync" -> set bottle.settings.enhancedSync = .esync
      - "enable-controller-compat" -> set bottle.settings.inputConfig controller compat
      - "install-winetricks-verb" -> delegate to Winetricks (async, mark as non-reversible)
      - "install-dependency" -> delegate to DependencyManager (async, non-reversible)
      - "run-enhanced-diagnostics" -> set WINEDEBUG preset
      - "restart-wineserver" -> kill wineserver via Wine.killBottle (non-reversible)
      Settings changes are immediate writes via BottleSettings didSet auto-save. Return FixAttempt with .applied result, before/after values.

    - `@MainActor static func undo(attempt: FixAttempt, bottle: Bottle, program: Program?) -> Bool` -- Reverses a fix if reversible. Uses attempt.fixId and attempt.beforeValue to restore the previous state. Returns true if undo succeeded, false if the fix is non-reversible (winetricks, dependency, restart). Per locked decision: clearly label non-reversible actions.

    **FixPreview struct** (nested or at module scope): `settingName: String`, `currentValue: String`, `newValue: String`, `scope: String`, `isReversible: Bool`.

    Note: Some fixes involve async operations (winetricks, Wine commands). For those, apply() should initiate the operation and return the FixAttempt immediately with .pending result. The engine's verify step confirms completion.

    GPL v3 header, public, all methods handle errors gracefully.
  </action>
