  <action>
    Create the Checks/ directory under WhiskyKit/Sources/WhiskyKit/Troubleshooting/.

    Each check is a public struct conforming to TroubleshootingCheck: Sendable. Each has a `let checkId: String` and `func run(params:context:) async -> CheckResult`. These are THIN WRAPPERS (5-30 lines per run method). Do NOT reimplement diagnostic logic -- delegate to existing primitives.

    **CrashLogCheck.swift** (checkId: "crash.log_classify"): Create CrashClassifier(), read log file from context.preflight.recentLogURL, classify. If no log -> .unknown. If no matches -> .pass. If matches -> .fail with evidence: primaryCategory, confidence, headline, matchCount. Follow research example closely.

    **GraphicsBackendCheck.swift** (checkId: "graphics.backend_is"): Read "expected" and "scope" from params. Get current backend from bottle settings via context (bottleURL-based lookup may need @MainActor bridge). Compare resolved backend against expected. Return .alreadyConfigured if explicitly set to expected, .pass if resolves to expected via .recommended, .fail otherwise. Evidence: current, expected, configured. Follow research example exactly.

    **DXVKSettingsCheck.swift** (checkId: "dxvk.settings_check"): Check specific DXVK setting from params (e.g., "async" -> check dxvkAsync on BottleDXVKConfig). Read bottle settings. Return .pass/.fail with evidence showing current value. For "hud" -> check if DXVK HUD is enabled.

    **AudioDriverCheck.swift** (checkId: "audio.driver_check"): Delegate to WineRegistryAudioProbe or directly read audio driver from Wine registry via Wine.readAudioDriver (from Phase 6). Compare against params["expected"]. Return .pass/.fail/.alreadyConfigured with evidence: currentDriver, expectedDriver.

    **AudioDeviceCheck.swift** (checkId: "audio.device_check"): Use preflight data for audio device name and transport type. Check if a default device is available. If checking sample rate (params["checkRate"]), compare against expected. Return .pass if device available, .fail if no device, .warning if Bluetooth with potential latency.

    **AudioTestCheck.swift** (checkId: "audio.test_play"): This wraps WineAudioTestProbe. Since the test exe may not exist (MinGW not available), return .unknown with evidence noting test unavailable. If available, delegate to probe and normalize result.

    **DependencyCheck.swift** (checkId: "dependency.check_missing"): Check for specific DLL files in the bottle prefix. Params should include "dlls" (comma-separated list). For each DLL, check existence in system32/syswow64 directories under the bottle's drive_c. Return .pass if all found, .fail with evidence listing missing DLLs.

    **WinetricksVerbCheck.swift** (checkId: "winetricks.verb_check"): Delegate to Winetricks.loadInstalledVerbs (from Phase 2). Params include "verbs" (comma-separated). Compare installed vs expected. Return .pass if all installed, .fail with evidence listing missing verbs, .alreadyConfigured if all present.

    All checks must handle errors gracefully -- catch exceptions, return .error outcome with evidence describing the failure. Never throw from run().
  </action>

Params can include "expected" launcher type. Return .pass with evidence showing detected launcher type, or .unknown if no launcher detected.

    **ProcessRunningCheck.swift** (checkId: "process.running_check"): Check if specific processes are running. Params: "process" (name to check). Delegate to ProcessRegistry for count. Return .pass if running (evidence: count), .fail if not running.

    **EnvironmentCheck.swift** (checkId: "env.check_var"): Check if a specific environment variable is set in the bottle/program environment. Params: "key" (env var name), optionally "expected" (expected value). Resolve via EnvironmentBuilder if accessible, or check BottleSettings/ProgramOverrides directly. Return .pass if matches expected (or if set when no expected specified), .fail otherwise.

    **RegistryValueCheck.swift** (checkId: "registry.value_check"): Wraps Wine.queryRegistryKey. Params: "key" (registry path), "valueName", optionally "expected". Query the registry and compare. Return .pass/.fail with evidence showing current value. Handle missing key as .unknown.

    **GameConfigAvailableCheck.swift** (checkId: "game.config_available"): Wraps GameMatcher. Use context program info to attempt a match. Return .pass with evidence (game name, rating, variant count) if a match is found, .fail if no match. This lets flows suggest "Apply known-good config" when a game DB entry exists.

    **SettingValueCheck.swift** (checkId: "setting.value_check"): Generic check for any BottleSettings property. Params: "setting" (property name like "enhancedSync", "graphicsBackend"), "expected" (expected value). Read the setting value using a switch on the setting name, compare against expected. Return .pass/.alreadyConfigured/.fail with evidence.

    **DiagnosticsEnhanceCheck.swift** (checkId: "diagnostics.can_enhance"): Check if enhanced diagnostics (WINEDEBUG preset) can be applied. Check if a WineDebugPreset is currently set or if the default is in use. Return .pass if enhanced diagnostics would provide additional info, .alreadyConfigured if already using a debug preset, .fail if not applicable.
