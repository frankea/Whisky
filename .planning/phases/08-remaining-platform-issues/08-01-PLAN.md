<objective>
Enhance launcher presets and macOS compatibility with structured fix metadata for provenance display.

Purpose: The three-tier launcher discovery model (per user decision) requires each fix to carry a human-readable reason so the UI can show "Applied because macOS >= 15.4" or "Steam: locale fix for steamwebhelper". Currently LauncherPresets returns flat env var dicts and MacOSCompatibility sets vars without recording why. This plan adds structured fix descriptions to both systems and extends EnvironmentBuilder provenance to carry reason strings.

Output: LauncherPresets with per-fix detail structs, MacOSCompatibility with version-scoped fix registry, EnvironmentBuilder provenance with reason field.
</objective>

2. Add a method `func fixDetails() -> [LauncherFixDetail]` on LauncherType that returns the same env vars as `environmentOverrides()` but wrapped in LauncherFixDetail structs with reason strings. Each fix gets a short reason derived from the existing inline comments.
3. Keep `environmentOverrides()` as-is for backward compatibility but mark it with a note that `fixDetails()` is the richer API.

In MacOSCompatibility.swift:
1. Add a public struct `MacOSFix: Sendable` with fields: `key: String`, `value: String`, `reason: String`, `appliesFrom: MacOSVersion` (the minimum version where this fix activates).
2. Refactor `applyMacOSCompatibilityFixes(to:)` to build fixes from a static `allFixes: [MacOSFix]` array. Each existing env var assignment becomes a MacOSFix entry with a reason string (use the existing inline comments as source). The method still mutates the environment dict but now iterates the fix array, filtering by `MacOSVersion.current >= fix.appliesFrom`.
3. Add a public static method `activeFixes() -> [MacOSFix]` that returns only fixes applicable to the current macOS version, for UI display.
4. Keep existing `MacOSVersion` struct and constants unchanged.

FixCategory enum should be at module scope (following ClipboardPolicy pattern from Phase 1).
