  <name>Task 1: Enhanced duplicate() with progress phases and naming convention</name>
  <files>
    Whisky/Extensions/Bottle+Extensions.swift
  </files>
  <action>
1. **Naming convention helper:** Add a static/free function `nextDuplicateName(baseName:existingNames:) -> String` that:
   - If no existing bottle named "{baseName} Copy", return "{baseName} Copy"
   - If "{baseName} Copy" exists, try "{baseName} Copy 2", "Copy 3", etc.
   - `existingNames` is `[String]` from `BottleVM.shared.bottles.map { $0.settings.name }`

2. **Enhanced duplicate() method:** Update the existing `duplicate(newName:)` to add progress reporting:
   - Add a `progress: ((DuplicationPhase) -> Void)?` callback parameter (default nil for backward compatibility).
   - Define `DuplicationPhase` enum: `.calculatingSize`, `.copying(bytesCopied: Int64, totalBytes: Int64)`, `.updatingMetadata`, `.finalizing`
   - Before copy: call `progress?(.calculatingSize)`. Calculate source directory size using `FileManager` recursive enumeration of `URLResourceValues.totalFileAllocatedSize` in the detached task.
   - During copy: Since `FileManager.copyItem` is atomic and doesn't provide callbacks, use a file-by-file recursive copy approach instead:
     - Enumerate source directory recursively
     - Copy each file individually, accumulating bytes copied
     - Call `progress?(.copying(bytesCopied:totalBytes:))` periodically (every 50 files or 50MB, whichever comes first)
   - After copy: call `progress?(.updatingMetadata)` before pin/blocklist rewriting
   - Final: call `progress?(.finalizing)` before `loadBottles()`
   - If file-by-file copy is too complex for V1, keep `FileManager.copyItem` but use progress phases without byte granularity: `.calculatingSize` -> `.copying(0, totalSize)` -> `.updatingMetadata` -> `.finalizing`. The progress bar will be indeterminate for the copy phase but show phase labels. Per Claude's discretion on progress bar implementation.

3. **Error handling and cleanup:** On copy failure, remove the partially created directory (`try? FileManager.default.removeItem(at: newBottleDir)`) before re-throwing. This prevents orphaned partial clones per user decision.

4. **Exclude transient artifacts:** After copy, before metadata update, delete:
   - Any `.log` files in the new bottle's logs directory (old diagnostic logs)
   - Diagnosis history sidecar files (`.diagnosis-history.plist`)
   - Temp files in prefix (if any)
   This gives the clone a clean start per user decision ("optionally exclude old logs/diagnostic history for clean clone").
  </action>

  <done>Duplicate method has progress callback, naming convention helper, cleanup of transient artifacts, and partial clone cleanup on failure.</done>

   - Pass the progress callback to `bottle.duplicate(newName:progress:)`:
