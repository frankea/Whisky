  <action>
    **FlowLoader.swift**: Define `FlowLoader` as a caseless enum (following PatternLoader/GameDBLoader pattern). Static methods:
    - `static func loadIndex() -> FlowIndex?` -- Load index.json from Bundle.module using `Bundle.module.url(forResource: "index", withExtension: "json")`. Decode with JSONDecoder. Log error and return nil on failure (soft-fail-in-release per research pitfall 1).
    - `static func loadFlow(fileName: String) -> FlowDefinition?` -- Load a specific flow file from Bundle.module. The resource name is the filename without extension.
    - `static func loadAllFlows() -> [String: FlowDefinition]` -- Load all flows referenced in the index, keyed by categoryId.
    - `static func loadFragments() -> [String: FlowDefinition]` -- Load all JSON files from the fragments directory. Since SPM .process() flattens directories (per 05-01 decision), use resource names like "dependency-install" and "export-escalation".
    - Use `Logger(subsystem: "com.isaacmarovitz.Whisky", category: "FlowLoader")` for debug/error logging.

    **FlowValidator.swift**: Define `FlowValidator` as a caseless enum. Static method:
    - `static func validate(flows: [String: FlowDefinition], fragments: [String: FlowDefinition]) -> [ValidationIssue]` -- Returns array of issues. Checks: (a) every `on` target nodeId resolves within same flow's nodes or in any fragment's nodes, (b) every `fragmentRef` resolves to a fragment key, (c) every entryNodeId exists in nodes, (d) no unreachable nodes (BFS from entryNodeId), (e) step counter: no path longer than 50 nodes without a user-interaction node (type .fix or .verify) to prevent infinite loops per research pitfall 4. `ValidationIssue` is a struct with `flowId: String`, `nodeId: String?`, `message: String`, `severity: Severity` (warning/error).
    - In debug builds (#if DEBUG), call validate after loading and assert on errors. In release, log warnings.

    **CheckRegistry.swift**: Define `CheckRegistry: @unchecked Sendable` as a final class. Properties: `private var checks: [String: any TroubleshootingCheck]`. Methods:
    - `init()` -- empty registry (registerDefaults will be called by Plan 04 once checks exist).
    - `func register(_ check: any TroubleshootingCheck)` -- adds to dictionary keyed by checkId.
    - `func run(checkId: String, params: [String: String], context: CheckContext) async -> CheckResult` -- looks up check, returns error result if not found (per research example), otherwise delegates to check.run(params:context:).
    - Add a `func registerDefaults()` method that is empty for now (placeholder for Plan 04 to fill in).

    **PreflightCollector.swift**: Define `PreflightCollector` as a caseless enum. Static method:
    - `@MainActor static func collect(bottle: Bottle, program: Program?) async -> PreflightData` -- Collects cheap state: bottle.url, bottle.settings.name, program?.url, program?.name, launcher type via LauncherDetection if available (otherwise nil), Wine.isWineserverRunning, ProcessRegistry.shared.getProcessCount, findRecentLog (scan program's log directory for most recent .log file), program?.settings.lastExitCode if available, AudioDeviceMonitor default device name/transport, resolved graphics backend from bottle.settings. Set collectedAt to Date().
    - Private helper `static func findRecentLog(for bottle: Bottle, program: Program?) -> URL?` -- scans the program's or bottle's log directory for the most recent .log file by modification date.
    - Note: Some properties (like AudioDeviceMonitor, LauncherDetection, ProcessRegistry) may not be directly accessible from WhiskyKit depending on target boundaries. If AudioDeviceMonitor or similar types are in the app target, use optional detection with nil fallback. The preflight is best-effort.

    All files get GPL v3 header. Follow caseless enum pattern for utility namespaces. Use os.log Logger.

  <action>
    Define `TroubleshootingFlowEngine` as `@MainActor public final class: ObservableObject`. This generalizes AudioTroubleshootingEngine from Phase 6 into a JSON-driven state machine.

    **Published state:**
    - `@Published public var session: TroubleshootingSession`
    - `@Published public var isRunningCheck: Bool = false`
    - `@Published public var currentNode: FlowStepNode?`
    - `@Published public var pathChanged: Bool = false` (triggers "Why path changed" UI)
    - `@Published public var pathChangeReason: String?`

    **Dependencies (injected via init):**
    - `flowDefinitions: [String: FlowDefinition]`
    - `fragments: [String: FlowDefinition]`
    - `checkRegistry: CheckRegistry`
    - `sessionStore: TroubleshootingSessionStore` (protocol or struct, defined minimally here as a protocol `TroubleshootingSessionStoring` with `func save(_ session: TroubleshootingSession)` and `func completeSession(_ session: TroubleshootingSession)` -- concrete implementation in Plan 05)
    - `private let logger = Logger(subsystem: "com.isaacmarovitz.Whisky", category: "TroubleshootingFlowEngine")`

    **Public methods:**
    - `init(flowDefinitions:fragments:checkRegistry:sessionStore:session:)` -- Takes all dependencies. Default session is a new TroubleshootingSession.
    - `func selectCategory(_ category: SymptomCategory)` -- Looks up flow by category.flowFileName in flowDefinitions. Sets session.symptomCategory, session.currentFlowCategoryId, session.phase = .checks. Calls navigateToNode(flow.entryNodeId). If flow not found, escalate.
    - `func navigateToNode(_ nodeId: String)` -- Resolves node via resolveNode(). If nil, escalate. Otherwise: set currentNode, call session.pushStep(node), auto-save. If node has checkId, call runCheck(). Track step count for cycle protection (50 automated steps without user interaction -> escalate per research pitfall 4).
    - `func runCheck(checkId:params:node:)` -- Private. Sets isRunningCheck = true. Creates CheckContext from session. Dispatches Task to call checkRegistry.run(). On completion: isRunningCheck = false, records result in session, determines next node from node.on map by outcome.rawValue, navigates to next. If outcome not in on map, try "default" key. Auto-saves.
    - `func applyFix(fixId:beforeValue:afterValue:)` -- Creates FixAttempt with .pending result, records in session, sets session.phase = .verify, auto-saves.
    - `func confirmFixApplied(fixId:)` -- Updates the pending FixAttempt to .applied.
    - `func userReportsFixed()` -- Sets session.outcome = .resolved, phase = .export, auto-saves, calls sessionStore.completeSession().
    - `func userReportsNotFixed()` -- If failedFixCount >= 3, escalate. Otherwise navigate to next fix or re-branch from current check.
    - `func skipStep()` -- Navigates to "skipped" or "default" target from current node's on map. Per locked "skip for now" decision.
    - `func goBack()` -- Restores previous step from stepHistory if available. Pops last entry, sets currentNode and session state accordingly.
    - `func escalate()` -- Sets session.phase = .escalation. Navigates to the export-escalation fragment's entry node if available.
    - `func startOver()` -- Resets session to fresh state, keeps bottle/program URLs. Phase = .symptom.
    - `func undoLastFix()` -- Finds the most recent applied FixAttempt, marks it .undone. This records intent; actual undo logic is in the FixApplicator (Plan 05).

    **Private methods:**
    - `resolveNode(_ nodeId: String) -> FlowStepNode?` -- Checks session's current flow first (from flowDefinitions keyed by currentFlowCategoryId), then searches all fragments.
    - `autoSave()` -- Calls `session.lastUpdatedAt = Date(); sessionStore.save(session)`.
    - `handleBranchChange(from oldPath:, to newPath:, reason:)` -- Sets pathChanged = true, pathChangeReason to the reason string. Per locked "Why path changed" decision.

    **Cycle protection:** Maintain a private `var automatedStepCount: Int = 0`. Increment on each navigateToNode call. Reset to 0 when a .fix or .verify type node is reached (requires user interaction). If automatedStepCount > 50, log warning and call escalate().

    **Session phase transitions:** The engine updates session.phase based on node.phase values as it navigates. When a node's phase differs from the previous node's phase, update session.phase.

    The engine has NO SwiftUI imports per research anti-pattern guidance. It lives in WhiskyKit.
  </action>
